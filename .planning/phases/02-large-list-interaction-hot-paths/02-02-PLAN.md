<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Inventory state to ingredient row summaries | Exact stock and expiry calculations come from persisted personal inventory data. |
| Shopping-list row action to reducer dispatch | Checklist generation can mutate shopping-list state and may be expensive on large seeded data. |
| Deferred modal shell to final content | Users may see a spinner before primary content, but final values must remain exact. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-T | Tampering | Ingredient stock and shopping-list checklist values | mitigate | Preserve `InventoryHelper` and `generateIngredient` exact calculations; do not replace them with approximate row values. |
| T-02-02-R | Repudiation | Performance evidence for cross-list fixes | mitigate | Write interaction ids for ingredient and shopping-list checks so summaries can trace which list failed. |
| T-02-02-D | Denial of Service | Shopping-list checklist generation before modal open | mitigate | Open the shell first, then schedule generation work after visible feedback is present. |
</threat_model>
