<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry to local dev environment | Adding `@playwright/test` introduces package supply-chain trust. |
| Browser test runner to app localStorage | Playwright fixtures write persisted Redux state before app startup. |
| Local command environment to test process | Test-control env vars influence network and diagnostic behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-SC | Tampering | `@playwright/test` dependency | mitigate | Verify package metadata with `npm view @playwright/test name version license --json` before adding the dependency; stop if metadata is unavailable or package name differs. |
| T-01-01-I | Information Disclosure | `playwright.config.ts` and scripts | mitigate | Do not read or print `.env` secret values; use only non-secret test-control env vars. |
| T-01-01-D | Denial of Service | Playwright webServer startup | mitigate | Use `reuseExistingServer: true`, configurable `E2E_PORT`, and list-mode verification before running full browser suites. |
</threat_model>
